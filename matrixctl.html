adduser¶

matrixctl.adduser.adduser(arg)[source]¶

Add a User to the synapse instance.

It runs ask_password() first. If ask_password() returns None it generates a password with gen_password(). Then it gives the user a overview of the username, password and if the new user should be generated as admin (if you added the --admin argument). Next, it asks a question, if the entered values are correct with the ask_question function.

If the ask_question function returns True, it continues. If not, it starts from the beginning.

Depending on the --ansible switch it runs the adduser command via ansible or the API

matrixctl.user.user(arg)[source]¶

List information about an registered user.

It uses the admin API to get a python dictionary with the information. The generate_user_tables function makes the information human readable. The Python package tabulate renders the table as shown below, if everything works well.

$ matrixctl user dwight
User:
+----------------------------+--------------------------------------+
| Name                       | dwight                               |
| Password Hash              | $2b$12$9DUNderm1ffL1NincPap3RC       |
|                            | ompaNY1725.slOUghAvEnu5cranT0n       |
| Guest                      | False                                |
| Admin                      | True                                 |
| Consent Version            |                                      |
| Consent Server Notice Sent |                                      |
| Appservice Id              |                                      |
| Creation Ts                | 2020-04-14 13:04:21                  |
| User Type                  |                                      |
| Deactivated                | False                                |
| Displayname                | Dwight Schrute                       |
| Avatar Url                 | mxc://dunder-mifflin.com/sCr4        |
|                            | nt0nsr4ng13rW45Cr33d                 |
+----------------------------+--------------------------------------+

Threepid:
+--------------+-----------------------------------+
| Medium       | email                             |
| Address      | dwight_schrute@dunder-mifflin.com |
| Validated At | 2020-04-14 15:30:21.123000        |
| Added At     | 2020-04-14 15:29:19.100000        |
+--------------+-----------------------------------+

If the user does not exist, the return looks like:

$ matrixctl user mose
2020-04-14 13:58:13 - ERROR - The request was not successful.
2020-04-14 13:58:13 - ERROR - There is no user with that username.

matrixctl.users.users(arg)[source]¶

Print a table of the matrix users.

This function generates and prints a table of matrix user accounts. The table can be modified.

  • If you want guests in the table use the --with-guests switch.

  • If you want deactivated user in the table use the --with-deactivated switch.

Example

matrixctl.password_helpers.ask_password()[source]¶

Ask the user to create a password.

The user will be asked twice, after that, the function compares the two entered passwords. If they are the same, the function will return the password.

Note

If the user presses enter twice, without entering a password, the function will return None. This is by-design.

Return type

Optional[str]

Returns

Returns the user entered password or None

matrixctl.password_helpers.ask_question(question='Is everything correct?')[source]¶

Asks the user a simple yes/no a question.

The answer will be returned as bool. - True: if the user answered y / j (j for the German yes: Ja) - False: if the user answers n

If the user answerd with an invalid answer (not y / j / n) the function asks again.

Parameters

question (str) – The yes/no question the user should be asked

Return type

bool

Returns

True if the answer was y / j and False if the answer was n

matrixctl.password_helpers.gen_password(length=16, min_digits=3, min_special=1, max_special=3)[source]¶

Generate a password.

It uses lower- and uppercase characters, digits and some special characters, you can find in the SPECIAL variable in this file.

It uses at least:

  • min_digits digits

  • min_special special characters

It uses maximal:

  • max_special special characters

The rest are lower- and uppercase characters.

Parameters

  • length (int) – The desired length of the password to generate

  • min_digits (int) – The desired minimum of digits of the password

  • min_special (int) – The desired minimum of special characters of the password

  • max_special (int) – The desired maximum of special characters of the password

Return type

str

Returns

It returns the generated password
